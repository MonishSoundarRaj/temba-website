MS-Temba: Multi-scale Temporal Mamba for Efficient Action Detection in Long Videos

Arkaprava Sinha, Monish Soundar Raj, Pu Wang, Ahmed Helmy, Srijan Das
UNC Charlotte
arXiv 2024
Paper Code arXiv

MS-Temba achieves state-of-the-art performance on long-video datasets and matches the best-performing method on short-video datasets, using 88.5% fewer parameters. The model's efficiency increases with video duration, making it ideal for processing hour-long videos.

Abstract

Action detection in real-world scenarios is particularly challenging due to densely distributed actions in hour-long untrimmed videos. It requires modeling both short- and long-term temporal relationships while handling significant intra-class temporal variations. Previous state-of-the-art (SOTA) Transformer-based architectures, though effective, are impractical for real-world deployment due to their high parameter count, GPU memory usage, and limited throughput.

We propose Multi-scale Temporal Mamba (MS-Temba), comprising two key components: Temporal Mamba (Temba) Blocks and the Temporal Mamba Fuser. Temba Blocks include the Temporal Local Module (TLM) for short-range temporal modeling and the Dilated Temporal SSM (DTS) for long-range dependencies. By introducing dilations, a novel concept for Mamba, TLM and DTS capture local and global features at multiple scales.

MS-Temba is validated on three public datasets, outperforming SOTA methods on long videos and matching prior methods on short videos while using only one-eighth of the parameters.

Key Contributions

Novel Architecture

Introduction of MS-Temba, a novel Mamba-based architecture tailored for densely labeled Temporal Action Detection, achieving state-of-the-art performance with significantly fewer parameters.

Multi-scale Temporal Modeling

Exploration of different temporal scales through novel components in Temba block, especially dilated Temporal SSMs, enabling effective capture of both local and global features.

Efficiency Breakthrough

Achievement of state-of-the-art performance while using 88% fewer parameters than previous methods, making it practical for real-world deployment.
